# Project 1d1  

**Due**: Monday, Sept 22  
**What**: Start thinking about proj2  
**What to do**:  
  
Reflection piece. What have you learned? Based on the above experience, what tools are needed surrounding the LLM in order to make it useful for requirements amplification and condensation. If you were designing a start up to do tools to help RE with LLMs, what tools would that start up sell? Note where we are after the tools you will invent for proj2,3 that use LLMs as a sub-routine to support engineering requirements.

**What to hand in**:  
  - Reflection document (max 2 pages) addressing the following questions:

1. **What are the pain points in using LLMs?**
2. **Any surprises? Eg different conclusions from LLMs?**
3. **What worked best?**
4. **What worked worst?**
5. **What pre-post processing was useful for structuring the import prompts, then summarizing the output?**
6. **Did you find any best/worst prompting strategies?**
